# MJ Prompt Studio ユーザーマニュアル

MJ Prompt Studioは、画像生成に使うプロンプトを考え、整え、比較し、結果を見直すための制作支援アプリです。画像生成サービスを自動操作するアプリではありません。最終プロンプトは自分でコピーし、生成結果の画像も自分で取り込みます。

## はじめに

最初に使うときは、次の流れだけ覚えておけば十分です。

1. `Composer`で、作りたい画像の内容を日本語でざっくり書く。
2. `AIで構成案を作る`を押して、プロンプトの部品に分ける。
3. 必要な行を直し、`Compile`で最終プロンプトを作る。
4. `コピー`でプロンプトをコピーし、画像生成サービスへ自分で貼り付ける。
5. 生成された画像を保存し、`Result Review`へ取り込んで見直す。

画面は大きく4つに分かれています。

- 左側: プロジェクト、プロンプト履歴、よく使う操作。
- 中央: `Composer`、`Free Editor`、`Matrix Lab`、`Reference Library`、`Result Review`、`Settings`の作業画面。
- 右側: AIの要約、パラメータ提案、プロンプト診断。
- 下側: AI処理の進行状況、キャンセル、再実行、検証結果。

## まず設定したいこと

### AI機能を使える状態にする

AI支援を使う場合は、右上または中央タブの`Settings`を開きます。

1. `Settings`を開く。
2. `APIキー`欄に利用するキーを入力する。
3. `APIキーをこのセッションに適用`を押す。
4. `接続テスト`を押し、接続状態が`有効`になることを確認する。

保存しておきたい場合は`設定を保存`を押します。保存できない環境では、その起動中だけ有効になります。

### Privacy modeを使う

入力内容や画像の扱いを控えめにしたいときは、`Settings`の`Privacy mode`を有効にして`設定を保存`します。AI分析を実行する場合、分析対象の画像や必要な説明はAIへ送られます。送信したくない画像は、AI分析やAIレビューを実行しないでください。

## プロジェクトを始める

### 新しい制作を始めたい

1. 上部Toolbarの`新規プロジェクト`を押す。
2. プロジェクト名を入力する。
3. 左側のプロジェクト一覧に新しい項目が表示されることを確認する。

ホテル広告、商品写真、建築、キャラクターなど、制作テーマごとにプロジェクトを分けると後から探しやすくなります。

### 以前の制作を開きたい

1. 上部Toolbarの`開く`を押す。
2. 開きたいプロジェクトを選ぶ。
3. 左側の`プロンプト履歴`で過去の保存内容を確認する。

### 作業を保存したい

上部Toolbarの`保存`を押します。保存すると、左側の`プロンプト履歴`に記録が追加されます。作業を戻したいときはToolbarの`Undo`、戻しすぎたときは`Redo`を使います。

## ざっくりした日本語からプロンプトを作る

`Composer`は、いちばん基本の作業画面です。

### 何を作りたいかを伝える

1. 中央の`Composer`タブを開く。
2. `AI Brief`に、作りたい画像を日本語で書く。
3. `AIで構成案を作る`を押す。
4. `プロンプト構成ブロック`に内容が入るのを確認する。

例:

```text
高級ホテルの朝食広告。朝の自然光で静か。パンとコーヒーを中心に、人物は写さない。余白は多め。
```

最初から完璧に書く必要はありません。目的、被写体、雰囲気、避けたいものを短く書けば十分です。

### ブロックを直す

`プロンプト構成ブロック`には、被写体、環境、構図、カメラ、照明、質感、色、スタイルなどが行ごとに並びます。

- 被写体を変えたい: `Subject`を直す。
- 明るさや光を変えたい: `Lighting`を直す。
- 画角や配置を変えたい: `Composition`や`Camera / Lens`を直す。
- 質感を強めたい: `Material / Texture`を直す。
- 色味を整えたい: `Color Palette`を直す。
- 文字を入れたい: `Text in Image`に短い文言を入れる。
- 避けたい表現を肯定的に指定したい: `Positive Constraints`を使う。

各行の右側にはAI補助があります。

- `AI補完`: 足りない言葉を補う。
- `候補`: 使えそうな表現を出す。
- `専門語化`: 曖昧な日本語を画像向けの英語表現にする。
- `短縮`: 長すぎる表現を短くする。
- `説明`: その行の意味や改善方向を確認する。

AIが出した変更は、勝手には上書きされません。変更前、変更後、理由を確認してから適用します。

### 最終プロンプトを作る

1. ブロックを編集する。
2. `Compile`を押す。
3. `Compiled Prompt`を確認する。
4. 問題なければ`コピー`を押す。
5. 画像生成サービスへ自分で貼り付ける。

コピー前に右側の`Prompt Doctor`を確認し、警告が出ていれば修正してから使うと安定します。

## 英語プロンプトを直接整える

すでにプロンプト案がある場合は、`Free Editor`が便利です。

### 日本語メモから英語へ整えたい

1. `Free Editor`を開く。
2. 左の欄に日本語のメモや指示を書く。
3. 上部の`語彙強化`、`精密化`、`構造化`などを押す。
4. 右の欄に変換結果が入る。
5. 下のライブプレビューで全体を確認する。

### 目的別のボタン

- `語彙強化`: ありきたりな言葉を豊かにする。
- `精密化`: 被写体、光、構図を具体化する。
- `短縮`: 長いプロンプトを短くする。
- `スタイル変換`: 雰囲気や仕上げを変える。
- `構造化`: 文章をプロンプト向けの並びに整える。
- `ルールセット修正`: 使いやすい形に整える。

## パラメータを決める

右側の`Parameter Advisor`では、画像の比率、スタイルの強さ、多様性、固定用の値などを調整できます。

### 自分で調整する

1. 右側の`Parameter Advisor`を見る。
2. 必要な項目を変更する。
3. `Parameter Rulesを適用`を押す。
4. `Compiled Prompt`に反映された内容を確認する。

### AIに目的別で提案してもらう

1. `Parameter Advisor`上部の目的欄に、狙いを書く。
2. `AIで目的別提案`を押す。
3. 提案内容と理由を確認する。
4. 問題なければ適用する。

例:

- 精密に狙いたい。
- 作風をいくつか試したい。
- 商品の形は崩したくない。
- 文字を入れたいので安定させたい。

## プロンプトを診断する

右側の`Prompt Doctor`は、プロンプトの弱点を確認する場所です。

### いつ使うか

- 結果が毎回ばらつく。
- 被写体が安定しない。
- 雰囲気は良いが、商品や素材が弱い。
- 否定したい内容がうまく伝わらない。
- パラメータを変えたら狙いから外れた。

### 使い方

1. `Composer`で現在の内容を整える。
2. 右側の`Prompt Doctor`で`AIで改善案を生成`を押す。
3. エラー、警告、情報を確認する。
4. `AI Fix Candidates`から適用したい案を選ぶ。
5. 差分を確認してから適用する。

警告が残っていても、意図的な表現ならそのまま進めて構いません。

## 参考画像を管理する

`Reference Library`は、参考画像や雰囲気の素材を集めて、プロンプトに使う言葉を取り出す画面です。

### 参考画像を追加したい

1. `Reference Library`を開く。
2. `参照画像を追加`を押して画像を選ぶ。画像を画面へドラッグ&ドロップしても追加できます。
3. 一覧から画像を選び、右側のプレビューを確認する。

画像サイズ、形式、カラーパレットの目安も表示されます。

### 画像から表現を取り出したい

1. 画像を選ぶ。
2. `AI分析`を押す。
3. 照明、色、構図、質感、推奨モード、抽出語彙を確認する。
4. 使いたい場合は`語彙を使う`を押す。
5. Composerへ戻される変更案を確認し、必要なら適用する。

### 素材を整理したい

- 検索欄でタイトル、タグ、抽出語彙を探す。
- フィルターで`Image Reference`、`Style Reference`、`Moodboard`などを絞り込む。
- `タグ保存`で素材にタグを付ける。
- 不要な素材は`削除`を押し、確認してから削除する。

## 実験パターンを作る

`Matrix Lab`は、少しずつ条件を変えたプロンプトをまとめて作る画面です。

### 使うとよい場面

- どの色味が合うか比較したい。
- スタイルの強さを段階的に試したい。
- 構図だけを変えて見比べたい。
- 固定条件を守りながら複数案を作りたい。

### 実験を作る

1. `Matrix Lab`を開く。
2. 実験目的を書く。
3. `AIで実験計画を作る`を押す。
4. 固定条件、可変軸、評価観点を確認する。
5. `Variant生成`を押す。
6. 一覧から使いたいVariantを選んでコピーする。

必要に応じて、一括コピーやCSV/Markdown出力も使えます。

## 生成結果を見直す

`Result Review`は、実際に生成した画像を取り込み、元プロンプトと比べて改善点を見つける画面です。

### 画像を取り込む

1. 画像生成サービス側で作った画像をローカルに保存する。
2. `Result Review`を開く。
3. `画像を取り込む`を押して画像を選ぶ。
4. 画像プレビュー、Source Prompt、Parameters Snapshotを確認する。

### AIレビューを実行する

1. 取り込んだ画像を選ぶ。
2. `AIレビュー`を押す。
3. スコア、良い点、課題、次のプロンプト候補を確認する。
4. 改善候補を使う場合は`次プロンプト作成`を押す。
5. Composerに戻る変更案を確認し、必要なら適用する。

### 複数結果を比べる

1. 複数の画像を取り込む。
2. それぞれAIレビューを実行する。
3. `比較`を押す。
4. どの案が商用利用、構図、スタイル一致などに向いているかを見る。

## コピーとエクスポート

### すぐ使うプロンプトだけ欲しい

`Composer`の`コピー`を押します。Compiled Promptだけがクリップボードに入ります。

### 制作ログとして残したい

上部Toolbarの`エクスポート`を押し、用途に合う形式を選びます。

- `Prompt only`: 最終プロンプトだけ保存する。
- `Prompt + manual checklist`: 手動投入時に確認する項目も一緒に保存する。
- `Markdown record`: 制作ログとして読みやすく保存する。
- `JSON project snapshot`: プロジェクト状態を保管する。
- `CSV matrix variants`: Matrix Labの一覧を表計算で扱う。
- `Markdown matrix variants`: Matrix Labの実験内容を文書で共有する。

## 下部のJobsを見る

AI処理を実行すると、下部の`Jobs`に状況が表示されます。

- `queued`: 順番待ち。
- `running`: 処理中。
- `succeeded`: 完了。
- `failed`: 失敗。
- `cancelled`: キャンセル済み。

時間がかかりすぎる場合はキャンセルできます。失敗した場合は、内容を確認して再実行できます。

## よくある目的別ガイド

### 高品質な商品写真風にしたい

1. `Composer`のAI Briefに、商品、背景、光、使いたい雰囲気を書く。
2. `Lighting`と`Material / Texture`を具体化する。
3. `Parameter Advisor`で「商品の形は崩したくない」と入力して提案を受ける。
4. `Prompt Doctor`で弱い表現を確認する。
5. 生成結果を`Result Review`で見直す。

### 参考画像の雰囲気を取り入れたい

1. `Reference Library`に画像を追加する。
2. `AI分析`を実行する。
3. 抽出語彙を確認する。
4. `語彙を使う`でComposerへ戻す。
5. `Style`、`Lighting`、`Color Palette`に反映されているか確認する。

### 何パターンか試して比較したい

1. まず`Composer`で基準プロンプトを作る。
2. `Matrix Lab`を開く。
3. 変えたい条件を目的欄に書く。
4. 実験計画を作り、Variantを生成する。
5. 使いたいVariantをコピーして生成する。
6. 生成画像を`Result Review`へ取り込み、比較する。

### 結果が狙いからずれる

1. `Result Review`で画像を取り込む。
2. `AIレビュー`を押す。
3. 課題と次のプロンプト候補を見る。
4. `次プロンプト作成`でComposerへ戻す。
5. `Prompt Doctor`で矛盾や弱い指定を確認する。

## 困ったとき

### AIが動かない

- `Settings`で接続状態を確認する。
- APIキーを入力して`APIキーをこのセッションに適用`を押す。
- `接続テスト`を実行する。
- 下部の`Jobs`で失敗理由を見る。

### 変更が思った内容と違った

- すぐにToolbarの`Undo`を押す。
- AI Patchは適用前に差分を確認し、迷う場合は適用しない。
- 保存済みの履歴は左側の`プロンプト履歴`で確認する。

### 参考画像が表示されない

- 画像ファイルがローカルに存在するか確認する。
- 画像をもう一度`参照画像を追加`から取り込む。
- プレビューが出なくても、タグやメモで管理できる場合があります。

### プロンプトが長すぎる

- 各Blockの`短縮`を使う。
- `Free Editor`で`短縮`を押す。
- 重要度の低い`Notes`や重複した形容を減らす。

### 生成結果が安定しない

- `Parameter Advisor`で精密さを優先した提案を受ける。
- `Composition`、`Camera / Lens`、`Lighting`を具体的にする。
- `Matrix Lab`で少しずつ条件を変えて比較する。

## 安全に使うための注意

- このアプリは画像生成サービスへ自動投稿しません。
- コピー、貼り付け、画像アップロードは自分で行います。
- 参照画像や生成結果画像は、AI分析やAIレビューを実行したときだけ分析対象になります。
- 機密性の高い画像や文章は、AI分析に送ってよいか確認してから使ってください。
